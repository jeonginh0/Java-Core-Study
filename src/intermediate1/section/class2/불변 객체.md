# 불변 객체
## 기본형과 참조형의 공유
자바의 데이터 타입을 가장 크게 보면 기본형(Primitive Type), 참조형(Reference Type)으로 나눌 수 있다.
- **기본형**: 하나의 값을 여러 변수에서 절대로 공유하지 않는다.
- **참조형**: 하나의 객체를 참조값을 통해 여러 변수에서 공유할 수 있다.

## 공유 참조와 사이드 이펙트
사이드 이펙트(Side Effect)는 프로그래밍에서 어떤 계산이 주된 작업 외에 추가적인 부수 효과를 일으키는 것.
```java
b.setValue("부산"); //b의 값을 부산으로 변경해야 함.
System.out.println("부산 -> b");
System.out.println("a = " + a); //사이드 이펙트 발생
System.out.println("b = " + b);
```
- 개발자는 b의 주소값을 서울에서 부산으로 변경할 의도로 값 변경을 시도했다.
- 하지만 a,b는 같은 인스턴스를 참조한다. 따라서 a의 값도 함께 부산으로 변경되어 버린다.

프로그래밍에서 사이드 이펙트는 보통 부정적인 의미로 사용이 된다. 사이드 이펙트는 프로그램의 특정 부분에서 발생한 변경이 의도치 않게 다른부분에 영향을 미치는 경우에 발생한다.
이로 인해 디버깅이 어려워지고 코드의 안정성이 저하될 수 있다. 

## 사이드 이펙트 해결 방안
a와 b가 처음부터 다른 인스턴스를 참조하면 됨.
```java
Address a = new Address("서울");
Address b = new Address("부산");
```
